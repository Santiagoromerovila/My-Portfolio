Design and Development of a Dynamic Personal Web Portfolio

July, 2025 · 1034 words · Santiago Bartolomé Romero Vila

This project involves the design, coding, and evolution of a personal portfolio website developed as a professional platform. The site was built from scratch using HTML, CSS, and JavaScript after completing an introductory web development course offered by IBM on Coursera. This website serves both as a professional showcase and a continuous learning environment, applying key concepts in accessibility, responsive design, interface management, and user experience.

Introduction

Personal digital presence has become a strategic tool for both students and professionals. Beyond platforms like LinkedIn or GitHub, having a personal website allows full control over content presentation, visual style, and professional narrative. With this in mind, I decided to develop a personal website from scratch to integrate knowledge acquired through HTML and CSS training and to progressively expand it as my web development skills grow.

This portfolio serves a dual purpose:

  • Professional showcase: To display my resume, technical projects, work experience, and achievements.
  • Experimental environment: To apply, adapt, and continuously practice new web technologies and development methodologies.

Training and Technological Approach

The website development began after completing a foundational HTML, CSS, and JavaScript course offered by IBM. This course provided the fundamentals for understanding the semantic structure of a web page, the separation of content (HTML) and presentation (CSS), as well as key principles such as:

  • Responsive design.
  • Best practices in accessibility.
  • Logical separation between structure, style, and functionality.

From there, the website was enriched with animations, dark mode, dynamic buttons, intuitive navigation, and modern design practices (flat design, safe web typography, visual hierarchy, etc.).

Website Structure

The site is divided into several interconnected sections, navigable through a persistent navigation bar:

  • Home / About Me:Personal introduction, interests, and key skills.
  • Resume:Education, languages, technical skills, and experience.
  • Projects:Detailed project descriptions with images, videos, and technical write-ups.
  • Work: Highlights of professional experience.
  • Press:Media appearances or featured events.

Each section is designed with a clean, content-centered approach and optimized for both desktop and mobile devices.

Technical Features

Semantic HTML

A rigorous use of semantic tags like <header>, <nav>, <section>, <main>, <footer> was implemented to improve accessibility and optimize SEO. Relevant meta tags are also used for search engines and social media.

Visual Design with CSS

The visual design is centralized using CSS variables (:root), allowing easy switching of full color palettes (light/dark mode, backgrounds, text and link colors).

Example of theme definition:


                :root {
                 --bg-color: #f2f2f2;
                 --text-color: #333;
                 --nav-bg: #2c5364;
                --footer-bg: #0f2027;
                }

Dynamic Dark Mode

One JavaScript-powered feature is persistent dark mode, which stores user preferences in localStorage and updates CSS classes in real time:


                if (storedMode) setMode(storedMode);

The 🌙 / ☀️ toggle button allows users to switch modes without reloading the page.

Responsive Design

Through media queries, the site adapts properly to mobile, tablet, and large-screen devices. Elements like the profile image, navigation bar, or buttons are dynamically reorganized or scaled.


                @media (min-width: 768px) {
                .profile {
                    flex-direction: row;
                    justify-content: center;
                    text-align: left;
                }
                }

Multimedia and Visual Content

Each project includes scalable images, embedded videos, and highlighted code using highlight.js. Interactive social media icons are also integrated, enhancing professional connectivity.

Resource Integration and Best Practices

In addition to visual and structural design, modern web development practices have been implemented:

  • Accessible and secure links (rel="noopener noreferrer").
  • Multi-browser compatibility.
  • Reusable CSS styles organized by sections.
  • Scalable maintenance strategy: new pages can be easily integrated using shared templates.

Relevant GitHub repositories are also linked, providing visibility into source code and encouraging technical transparency.

Learning and Evolution

This project has been key in strengthening my knowledge of web development. While I do not intend to pursue a career specifically in this field, it serves as a solid foundation for future projects more focused on AI and Big Data. Beyond coding, this experience has helped me develop:

  • A structured mindset for UI/UX design.
  • Understanding of static website architecture.
  • Good practices in accessibility and usability.
  • Self-management and autonomous learning skills.

Each new feature - from dark mode to project organization - has been an opportunity to reinforce and apply learned concepts with independent judgment.

Conclusion and Future Work

Creating this personal portfolio has been more than a technical exercise - it has been an experience of consolidation and projection. Consolidation, because it allowed me to apply tangible skills from courses like the one from IBM. Projection, because it opens the door to future improvements such as:

  • Database integration (e.g., for contact forms or a blog).
  • Use of frameworks like React or Bootstrap.
  • Improved SEO and web analytics.
  • Migration to a CMS or integration with a lightweight backend (Node.js).

The site remains a living platform, constantly evolving - just like the professional career it represents.
