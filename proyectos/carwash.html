Embedded Systems Development with AVR-C and ATmega640

April, 2025 ¬∑ 636 words ¬∑ Santiago Bartolom√© Romero Vila

A low-level real-time control system for a fully automated car wash

Overview

In this project, I developed a complete embedded control system for an automated car wash installation, using the ATmega640 microcontroller and programming entirely in AVR-C. This was my first experience designing a full-scale, real-time control solution from the ground up. I used an AVR Dragon debugger to program and test the firmware in-system, ensuring correctness and safety across all subsystems.

Description

The goal was to create a robust embedded firmware that could coordinate multiple mechanical components in real-time, including motors, sensors, actuators, and signaling systems. The main components included:

  • Barrier gate with blinking warning light
  • Two vertical rollers and one horizontal roller
  • Fan-based drying unit
  • Conveyor chain system
  • Traffic light (red/yellow/green)
  • Car presence, reverse detection, and limit sensors

AVR-C: First Contact

This was my first time working with low-level AVR microcontrollers. I had to master:

  • Register-level configuration with DDRx, PORTx, and PINx
  • Using Timer0, Timer1, and Timer3 for delays, PWM, and timing logic
  • Interrupt handling for sensor feedback and emergency stops
  • Software debounce logic for mechanical sensors
  • PWM generation to control motor speeds
  • Hardware debugging with AVR Dragon

System Architecture

Entry Control

A barrier and blinking light managed car entry. A sensor detected the car, triggering the light, opening the barrier, and starting the conveyor.

Conveyor and Position Tracking

The conveyor was driven by a PWM-controlled motor. Timer-based counters simulated position tracking. Limit switches detected end-of-travel.

Washing Sequence

Vertical and horizontal rollers activated in sequence with precise timing. Motors were speed-controlled via PWM.

Drying and Exit

After washing, a fan system activated. A traffic light turned green once the car reached the final position. A reverse detection sensor halted the system if a car moved backwards unexpectedly.

Coordination and Safety

All actions were governed by a finite state machine. Each state corresponded to a stage in the washing process. Safety was enforced through:

  • Modular timer subsystems per component
  • Debounced sensor inputs
  • Emergency interrupts for critical sensors
  • Watchdog-like timeouts for stalled states

Challenges and Learnings

This project challenged me to think holistically about embedded control:

  • Synchronizing concurrent modules without an RTOS
  • Filtering noisy mechanical inputs
  • Ensuring real-time safety guarantees
  • Debugging live embedded systems

I learned to manage complexity while working directly with hardware and system timing.

Conclusion

This project taught me to think in terms of systems, not just code. I built a reliable, full-featured control application entirely in embedded C. It deepened my appreciation for low-level engineering, from sensors and PWM to real-time safety and hardware integration. This experience prepared me for more advanced roles in robotics, automation, and industrial embedded systems.

All the code can be found in my Github.
